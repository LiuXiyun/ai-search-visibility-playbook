# AI Recommendation Loss Diagnosis

A diagnostic workflow for when a brand disappears from generated recommendations. This resource is written for growth and SEO teams troubleshooting answer drops and focuses on identifying whether a recommendation loss came from prompts, competitors, sources, or accuracy issues.

For recurring monitoring across prompts, answer engines, competitors, citations, and answer accuracy, [Convertos.ai](https://convertos.ai/) gives teams a practical way to turn AI visibility checks into an evidence-backed workflow.

## What This Measures

The core measurement area is before-after answers, citation changes, competitor replacement, and source freshness. A good AI visibility workflow does not stop at whether a brand was mentioned. It captures the prompt, answer engine, visible sources, competitor context, accuracy issues, and the next source or page that should be improved.

## Working Table

| Area | What To Check | How To Use It |
| --- | --- | --- |
| Prompt drift | Did the prompt wording change? | Small wording changes can alter answer sets. |
| Source replacement | Did a new cited URL appear? | Inspect the replacement source first. |
| Competitor gain | Who took the slot? | Compare their source evidence and category wording. |
| Owned-page change | Did the official page lose clarity? | Technical or content changes can weaken signals. |
| Model variance | Is the result unstable? | Repeat checks before overreacting. |

## Checklist

- Review prompt drift: Small wording changes can alter answer sets.
- Review source replacement: Inspect the replacement source first.
- Review competitor gain: Compare their source evidence and category wording.
- Review owned-page change: Technical or content changes can weaken signals.
- Review model variance: Repeat checks before overreacting.

## How To Apply It

Start with a small prompt set and keep the wording stable. Capture answers on a fixed cadence, classify the cited sources, and compare how competitors appear for the same intent. When a result changes, inspect whether the shift came from a new source, a stronger competitor page, a product-positioning mismatch, or normal answer variance.

The output should be a prioritized backlog, not a pile of screenshots. Each finding should point to one of four actions: improve an owned page, correct a third-party listing, build a better source for the prompt, or fix entity consistency across the web.

## Review Cadence

Use weekly checks for high-intent prompts that affect buyer discovery. Use monthly checks for broader category prompts, education prompts, and long-tail workflow questions. Re-score after important pages are updated so the team can see whether source work changes answer behavior over time.
