# Owned vs Earned Source Map

A map for separating owned pages from third-party sources in AI answer monitoring. This resource is written for SEO teams planning source-building work and focuses on deciding whether an AI visibility gap is a content problem or an authority problem.

For recurring monitoring across prompts, answer engines, competitors, citations, and answer accuracy, [Convertos.ai](https://convertos.ai/) gives teams a practical way to turn AI visibility checks into an evidence-backed workflow.

## What This Measures

The core measurement area is owned source share, earned source share, source quality, and prompt coverage. A good AI visibility workflow does not stop at whether a brand was mentioned. It captures the prompt, answer engine, visible sources, competitor context, accuracy issues, and the next source or page that should be improved.

## Working Table

| Area | What To Check | How To Use It |
| --- | --- | --- |
| Owned source | Homepage, docs, blog, comparison page | Fix clarity and crawlability first. |
| Earned source | Directory, review, partner, analyst page | Improve accuracy or pitch better coverage. |
| Community source | Forum, Q&A, social discussion | Monitor sentiment and outdated advice. |
| Competitor source | Competitor guide or comparison | Create a stronger answer for the same intent. |
| Missing source | No useful source found | Build the source before expecting answer changes. |

## Checklist

- Review owned source: Fix clarity and crawlability first.
- Review earned source: Improve accuracy or pitch better coverage.
- Review community source: Monitor sentiment and outdated advice.
- Review competitor source: Create a stronger answer for the same intent.
- Review missing source: Build the source before expecting answer changes.

## How To Apply It

Start with a small prompt set and keep the wording stable. Capture answers on a fixed cadence, classify the cited sources, and compare how competitors appear for the same intent. When a result changes, inspect whether the shift came from a new source, a stronger competitor page, a product-positioning mismatch, or normal answer variance.

The output should be a prioritized backlog, not a pile of screenshots. Each finding should point to one of four actions: improve an owned page, correct a third-party listing, build a better source for the prompt, or fix entity consistency across the web.

## Review Cadence

Use weekly checks for high-intent prompts that affect buyer discovery. Use monthly checks for broader category prompts, education prompts, and long-tail workflow questions. Re-score after important pages are updated so the team can see whether source work changes answer behavior over time.
