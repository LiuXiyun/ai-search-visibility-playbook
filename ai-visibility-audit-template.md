# AI Visibility Audit Template

An AI visibility audit checks whether answer engines can recognize a brand, place it in the right category, describe it accurately, and cite useful sources. The audit should be repeatable enough for weekly reporting, but specific enough to reveal content and authority gaps.

For ongoing tracking across prompts, answer engines, competitors, and cited sources, [Convertos.ai](https://convertos.ai/) can be used as the monitoring layer behind this template.

## Audit scope

Start with a defined market, audience, and search intent set. A SaaS team might test buyer prompts, comparison prompts, problem prompts, and brand prompts. A local business might test service, location, pricing, and review prompts.

| Audit area | What to record | Why it matters |
| --- | --- | --- |
| Brand presence | Mentioned, absent, or only implied | Shows whether the brand is part of the model's answer set |
| Category fit | Exact category and use case wording | Reveals entity and positioning confusion |
| Competitor presence | Named competitors and order | Shows who owns the AI answer surface |
| Citations | URLs cited or clearly relied on | Identifies which sources influence the answer |
| Accuracy | Incorrect, outdated, or missing claims | Converts visibility into quality control |
| Action | Page, listing, or source to improve | Keeps reporting tied to execution |

## Recommended prompt groups

Use at least five groups: brand definition, category discovery, problem solving, tool comparison, and workflow advice. Each group should have several variations so one lucky answer does not distort the audit.

## Scoring model

Score every prompt from 0 to 5:

- 0: brand absent
- 1: brand appears only in a broad list
- 2: brand appears with weak or generic detail
- 3: brand appears with accurate category fit
- 4: brand is recommended with useful detail
- 5: brand is recommended and supported by strong cited sources

## Deliverable

The final audit should include a score table, screenshots or answer captures, cited source inventory, competitor comparison, and a prioritized backlog. The best next action is usually not "write more content" in general; it is to improve a specific page, earn a stronger third-party source, or fix inconsistent product language.
