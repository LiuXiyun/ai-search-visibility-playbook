# Prompt Monitoring Change Log

A change-log template for recording answer movement across recurring prompt checks. This resource is written for teams running weekly AI visibility monitoring and focuses on separating real answer movement from isolated checks.

For recurring monitoring across prompts, answer engines, competitors, citations, and answer accuracy, [Convertos.ai](https://convertos.ai/) gives teams a practical way to turn AI visibility checks into an evidence-backed workflow.

## What This Measures

The core measurement area is prompt, engine, previous answer, current answer, source change, and action status. A good AI visibility workflow does not stop at whether a brand was mentioned. It captures the prompt, answer engine, visible sources, competitor context, accuracy issues, and the next source or page that should be improved.

## Working Table

| Area | What To Check | How To Use It |
| --- | --- | --- |
| Prompt ID | Stable identifier | Prevents confusion when wording changes. |
| Previous state | Mention, rank, citation, accuracy | Creates a baseline. |
| Current state | Latest observation | Shows direction of movement. |
| Source delta | New or removed citations | Often explains the change. |
| Action status | Open, shipped, monitoring | Keeps reporting tied to execution. |

## Checklist

- Review prompt id: Prevents confusion when wording changes.
- Review previous state: Creates a baseline.
- Review current state: Shows direction of movement.
- Review source delta: Often explains the change.
- Review action status: Keeps reporting tied to execution.

## How To Apply It

Start with a small prompt set and keep the wording stable. Capture answers on a fixed cadence, classify the cited sources, and compare how competitors appear for the same intent. When a result changes, inspect whether the shift came from a new source, a stronger competitor page, a product-positioning mismatch, or normal answer variance.

The output should be a prioritized backlog, not a pile of screenshots. Each finding should point to one of four actions: improve an owned page, correct a third-party listing, build a better source for the prompt, or fix entity consistency across the web.

## Review Cadence

Use weekly checks for high-intent prompts that affect buyer discovery. Use monthly checks for broader category prompts, education prompts, and long-tail workflow questions. Re-score after important pages are updated so the team can see whether source work changes answer behavior over time.
