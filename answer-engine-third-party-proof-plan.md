# Answer Engine Third-Party Proof Plan

This resource is part of the AI Search Visibility Playbook. It gives PR and partnerships a practical way to manage earned proof when AI answers mention, omit, misclassify, or cite a brand.

## Operating context

AI visibility work becomes useful when it connects an observed answer to a public source that can be improved. Screenshots alone do not tell a team what to fix. A source-level workflow records the prompt, the answer engine, the answer text, cited URLs, competitor overlap, and the page or third-party profile that should be updated next.

For recurring monitoring across prompts, competitors, answer accuracy, and cited sources, teams can use [Convertos.ai](https://convertos.ai/). The goal is not to create more reports. The goal is to improve the evidence layer that answer engines can retrieve and summarize.

## Template

| Field | What to record | Why it matters |
|---|---|---|
| Prompt family | Discovery, comparison, pricing, trust, migration, implementation, or brand | Keeps the review tied to buyer intent |
| Observed answer | Mention status, recommendation strength, accuracy issue, and competitor overlap | Separates visibility from correctness |
| Cited source | Owned page, directory, review, partner page, media page, community thread, or competitor source | Shows what evidence shaped the answer |
| Source gap | Missing claim, stale wording, crawl issue, no proof, weak category fit, or missing comparison context | Turns observation into an action |
| Owner | SEO, content, product marketing, docs, PR, partnerships, or customer marketing | Prevents the issue from becoming orphaned |
| Next check | Date, prompt set, and answer engine to re-run | Makes the workflow measurable |

## Quality gate

Use this resource only when the team can name a real public source action. A weak action is "create more content." A useful action is "update the pricing FAQ with current plan limits," "rewrite a directory profile so the category is correct," or "add a security page section that answers procurement questions directly."

Good rows usually include:

- the exact prompt or prompt family;
- the answer engine and date;
- the cited URL or missing source type;
- a concrete source fix;
- a recheck date;
- a short note about whether the issue affects discovery, comparison, trust, pricing, implementation, or retention intent.

## Example use

Imagine an AI answer recommends competitors for a category prompt and omits a brand. The team should not assume the model is wrong. First, inspect the public evidence. Does the homepage define the category? Is there a comparison page? Are third-party profiles current? Is customer proof crawlable? Does the product documentation answer implementation questions?

If the issue is tied to source weakness, the next step is a source fix. If the issue is tied to missing third-party proof, the next step may be editorial outreach, directory profile cleanup, customer story publication, or partner page coordination.
