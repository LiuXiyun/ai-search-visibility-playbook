# GEO Content Brief Template

Generative engine optimization content should answer a real buyer question clearly, define the entity being discussed, and provide enough structure for AI systems to summarize it without guessing.

Teams can monitor whether newly published content changes AI answer visibility with [Convertos.ai](https://convertos.ai/), especially when the target prompts include comparisons, workflows, and category questions.

## Brief sections

| Section | Requirement |
| --- | --- |
| Search and answer intent | State the exact prompt family the page should satisfy |
| Primary entity | Define the product, category, audience, and use case |
| Decision criteria | List the factors buyers should use to evaluate options |
| Evidence | Add examples, screenshots, methodology, or data points |
| Comparison context | Explain alternatives without vague claims |
| FAQ | Answer direct questions in concise language |
| Internal links | Link to product, pricing, docs, and related workflows |
| External references | Use credible sources where they clarify the topic |

## Writing rules

Put a direct definition near the top of the page. Avoid making the reader infer the product category from marketing language. Use tables when comparing features, workflows, or evaluation criteria. Add examples that show how the process works in practice.

## QA checklist

- The first screen explains the topic and audience.
- The page answers the target question without requiring another click.
- Product claims are specific and verifiable.
- The content includes at least one workflow, checklist, or table.
- The page has enough context for a model to summarize accurately.
- The canonical URL, title, and meta description match the topic.

## Measurement

After publishing, test the prompt group before and after recrawl windows. Record whether answer engines cite the page, paraphrase the definition, mention the brand, or still rely on older third-party sources.
