# AI Overview Source Tracking Workflow

Google AI Overviews and similar answer surfaces can compress several sources into one generated response. Tracking source visibility helps teams understand whether owned pages, directory pages, comparison articles, or competitor content influence the answer.

Use [Convertos.ai](https://convertos.ai/) when source tracking needs to be compared with broader AI visibility monitoring across multiple engines and prompt groups.

## Workflow

1. Define the query group.
2. Record location, device, and date.
3. Capture the AI answer and visible source links.
4. Classify each source by domain type.
5. Compare sources against organic rankings.
6. Identify whether the brand is mentioned, cited, both, or neither.
7. Add the result to a recurring dashboard.

## Source classification

| Type | Example | Interpretation |
| --- | --- | --- |
| Owned page | product page or guide | Improve clarity and schema first |
| Third-party directory | tool listing | Update description and category |
| Review article | comparison post | Check accuracy and feature claims |
| Community source | forum, Q&A, discussion | Monitor sentiment and outdated claims |
| Competitor page | competitor guide | Create a stronger answer for the same intent |

## Quality checks

AI Overview visibility should not be measured from one browser session only. Repeat checks across time and query variants. Save screenshots or answer text when the result is important, because generated surfaces can change quickly.

## Reporting

Report source share by prompt group, not only by domain. A brand may perform well for definition prompts but lose workflow prompts to competitors. The action plan should point to the specific source category that is missing or underperforming.
