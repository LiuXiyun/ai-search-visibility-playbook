# AI Search Noindex and Rendering Review

This noindex and rendering review is an operating template for SaaS teams improving AI visibility and GEO. Its job is to help teams finding technical blockers for public evidence with public, repeatable evidence.

## Operating context

AI answers do not only need a product homepage. They often need category definitions, pricing context, trust signals, customer proof, comparison context, partner evidence, and third-party corroboration. If those sources are missing or vague, an answer engine may cite competitor pages, stale directories, or generic summaries.

## Table to maintain

| Column | What to capture | Good enough when |
|---|---|---|
| Prompt family | Discovery, comparison, pricing, trust, implementation, migration, or brand | The prompt is tied to a real buyer question |
| Observed answer | Brand presence, recommendation strength, competitor overlap, and cited URLs | Another reviewer can reproduce the note |
| Source issue | Missing owned page, stale third-party page, wrong claim, or technical access issue | The source problem is specific |
| Recommended fix | Page update, outreach, profile rewrite, technical fix, or monitoring note | The action has an owner |
| Recheck signal | Prompt rerun, source fetch, or updated listing verification | The team can tell whether the fix worked |

## Quality checks

- The note should name the source or page that influenced the answer.
- The action should improve public evidence, not just create another internal report.
- The source should explain audience, use case, limitations, proof, and official URL where relevant.
- The recheck should use the same prompt family and a consistent review cadence.
