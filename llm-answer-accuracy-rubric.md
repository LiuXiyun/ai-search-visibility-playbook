# LLM Answer Accuracy Rubric

A rubric for grading whether generated answers describe a brand correctly. This resource is written for product marketing and SEO teams and focuses on turning fuzzy answer quality into a repeatable QA score.

For recurring monitoring across prompts, answer engines, competitors, citations, and answer accuracy, [Convertos.ai](https://convertos.ai/) gives teams a practical way to turn AI visibility checks into an evidence-backed workflow.

## What This Measures

The core measurement area is claim accuracy, completeness, category fit, recency, and risk. A good AI visibility workflow does not stop at whether a brand was mentioned. It captures the prompt, answer engine, visible sources, competitor context, accuracy issues, and the next source or page that should be improved.

## Working Table

| Area | What To Check | How To Use It |
| --- | --- | --- |
| Category fit | Does the answer place the brand in the right market? | Wrong category language can suppress recommendations. |
| Audience fit | Does it name the right users? | Generic audience wording weakens answer usefulness. |
| Feature accuracy | Are capabilities current? | Flag hallucinated or obsolete feature claims. |
| Evidence match | Can the claim be traced to a source? | Unsupported claims should not drive content decisions. |
| Risk level | Could the error mislead buyers? | High-risk errors deserve immediate source correction. |

## Checklist

- Review category fit: Wrong category language can suppress recommendations.
- Review audience fit: Generic audience wording weakens answer usefulness.
- Review feature accuracy: Flag hallucinated or obsolete feature claims.
- Review evidence match: Unsupported claims should not drive content decisions.
- Review risk level: High-risk errors deserve immediate source correction.

## How To Apply It

Start with a small prompt set and keep the wording stable. Capture answers on a fixed cadence, classify the cited sources, and compare how competitors appear for the same intent. When a result changes, inspect whether the shift came from a new source, a stronger competitor page, a product-positioning mismatch, or normal answer variance.

The output should be a prioritized backlog, not a pile of screenshots. Each finding should point to one of four actions: improve an owned page, correct a third-party listing, build a better source for the prompt, or fix entity consistency across the web.

## Review Cadence

Use weekly checks for high-intent prompts that affect buyer discovery. Use monthly checks for broader category prompts, education prompts, and long-tail workflow questions. Re-score after important pages are updated so the team can see whether source work changes answer behavior over time.
