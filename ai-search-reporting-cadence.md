# AI Search Reporting Cadence

AI search reporting works best when it is consistent. One-off screenshots are useful for diagnosis, but recurring measurement is needed to see whether content, listings, and source improvements change answer behavior.

[Convertos.ai](https://convertos.ai/) is built around this recurring visibility workflow across prompts, engines, competitors, and citations.

## Weekly report

| Section | Question |
| --- | --- |
| Brand visibility | Did the brand appear for priority prompts? |
| Recommendation strength | Was the brand recommended or merely mentioned? |
| Competitors | Which competitors gained or lost visibility? |
| Citations | Which URLs supported the answers? |
| Accuracy | What claims were wrong or incomplete? |
| Actions | Which source or page should be improved next? |

## Monthly report

Monthly reports should focus on trend quality, not only counts. Compare prompt groups, engines, and competitor movement. Show which published changes may have influenced answer visibility and which gaps remain unresolved.

## Metrics

- Presence rate by prompt group
- Recommendation rate by engine
- Average position in list-style answers
- Citation share by domain
- Error rate in brand descriptions
- Competitor overlap
- Action backlog completion

## Operating rhythm

Use weekly reports for tactical fixes and monthly reports for strategy. If a prompt is commercially important, keep it in the weekly set. If it is exploratory or long-tail, move it to monthly monitoring until it proves important.
